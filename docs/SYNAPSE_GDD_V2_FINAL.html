Concepto

El jugador empieza con una sola neurona parpadeando en negro total. Sin UI. Sin tutorial. Solo un punto de luz que responde al toque. A medida que la mente crece, el juego se descubre a sí mismo. El primer prestige ocurre dentro de los primeros 20–25 minutos de juego activo.

¿Qué hace triunfar un idle game en mobile?

1. Cada sesión debe sentirse productiva

El jugador abre la app, ve progreso offline, toma 2–3 decisiones, y cierra sintiéndose más avanzado. SYNAPSE resuelve esto con el Sueño (pantalla de regreso), decisiones de upgrade, y el Disparo como momento satisfactorio.

2. La variedad mecánica sostiene la retención

Los idle que retienen 60+ días (Antimatter Dimensions, Cell to Singularity, Realm Grinder) introducen nuevas mecánicas con cada tier de prestige — no solo multiplicadores más grandes. SYNAPSE hace esto con las 3 Eras, el Sistema Focus, y las Mutaciones.

3. El prestige debe ser una inversión, no una pérdida

El momento de prestige debe sentirse como "ahora sí voy a romperla" — no como "perdí todo". La pantalla de Despertar muestra tiempo estimado del próximo ciclo y las mecánicas nuevas que se desbloquean.

4. Active play y idle play deben coexistir

El jugador que toca la pantalla activamente debe progresar más rápido que el que deja idle — pero el idle solo debe progresar significativamente. SYNAPSE usa el Sistema Focus para recompensar juego activo sin penalizar idle.

Arquitectura de loops — cada uno alimenta al siguiente

Loop 1 — Minutos

Tap → Pensamientos. Comprar neurona. Conexiones pasivas (+% por par de neuronas). Focus Bar se llena con taps activos → Insight burst (P3+). Carga de Disparo cada 20 min. Cascada = Disparo + Focus lleno = bonus masivo. Upgrade disponible.

Loop 2 — Horas

Idle acumula → unlock región. Nueva mecánica o fragmento narrativo. Ecos del Despertar (hints útiles). Barra de Consciencia sube. Llena → banner "DESPERTAR". Elegir Mutación para el próximo ciclo (P7+). Despertar manual.

Loop 3 — Días

Prestige → Patrones. Elegir Polaridad (P3+). Elegir Arquetipo (P5+). Elegir Mutación (P7+). Neural Pathways (P10+). Récord personal al batir tiempo. Challenge semanal. Cada Era cambia las reglas base.

Conexiones — simplificadas (pasivas, no burst/cooldown)

CAMBIO CLAVE vs v1

Las conexiones ya no tienen burst de 90s + cooldown de 60s. Son bonuses pasivos permanentes que se forman automáticamente entre neuronas de tipos diferentes. Cada par único da un bonus multiplicativo. El "momento activo" de dopamina se concentra exclusivamente en el Disparo y el Insight (Focus Bar).

// Cada par de tipos de neurona da un bonus pasivo // Ejemplo: Básica-Sensorial = +8%, Sensorial-Piramidal = +12% function calculateConnectionMult(state: GameState): number { const types = new Set(state.neurons.filter(n => n.owned > 0).map(n => n.type)) const pairs = (types.size * (types.size - 1)) / 2 return 1 + pairs * SYNAPSE_CONSTANTS.connectionBonusPerPair // 0.08 per pair } // Con 4 tipos activos = 6 pares × 0.08 = +48% bonus // Simple, predecible, satisfactorio al desbloquear nuevo tipo

POR QUÉ ESTO ES MEJOR

El viejo sistema tenía bursts de conexión (90s activo, 60s cooldown) + Cascada (burst + carga) + Gamma (activa todos los bursts). Eran 3 capas de complejidad para un resultado simple: "multiplicador temporal". Ahora: conexiones son pasivas (siempre activas), Disparo es el evento grande, y Cascada = Disparo + Focus lleno. Una interacción clara que el jugador entiende y optimiza.

Sistema Focus — tap que evoluciona en vez de morir

EL PROBLEMA QUE RESUELVE

En idle games, el tap se vuelve irrelevante en late-game porque la producción idle lo supera. Con 7 upgrades de tap, el jugador invierte en algo que no importa. El Sistema Focus hace que el tap cambie de propósito: en vez de generar thoughts directamente, llena una barra que dispara un burst poderoso.

Evolución del tap por fase

P1–P3: TAP DIRECTO

Tap genera thoughts directamente. tap = max(15, production × 0.3). Simple, intuitivo, satisfactorio para el onboarding. No hay Focus Bar todavía.

P4+: FOCUS BAR

Tap sigue generando thoughts (50% del valor original) PERO TAMBIÉN llena la Focus Bar. Al llegar al 100%, toque para activar Insight: ×5 producción por 20 segundos. Focus Bar se vacía y empieza de nuevo.

P10+: FOCUS MASTERY

Focus Bar tiene 3 niveles: Claro (100%), Profundo (200%), Trascendente (300%). Cada nivel de Insight es más poderoso pero requiere más taps. La decisión: ¿activo Insight rápido o espero al nivel más alto?

Mecánicas del Focus

// Focus Bar: 0.0 a 1.0 (nivel 1), 2.0 (nivel 2), 3.0 (nivel 3) const FOCUS_LEVELS = [ { threshold: 1.0, name: 'Claro', mult: 5, duration: 20 }, { threshold: 2.0, name: 'Profundo', mult: 12, duration: 15 }, { threshold: 3.0, name: 'Trascendente', mult: 25, duration: 10 }, ] function handleTap(state: GameState): GameState { if (state.prestigeCount < 3) { // Pre-P4: tap directo clásico const tapValue = Math.max(15, state.productionPerSecond * 0.3) state.thoughts += tapValue state.cycleGenerated += tapValue state.totalGenerated += tapValue } else { // P4+: tap da thoughts reducidos + llena Focus Bar const tapValue = Math.max(15, state.productionPerSecond * 0.15) state.thoughts += tapValue state.cycleGenerated += tapValue state.totalGenerated += tapValue // Llenar Focus Bar (rate ajustado por upgrades) state.focusBar += state.focusFillRate // base: 0.02 por tap // Post-P10: puede llenar hasta 3.0 const maxFocus = state.prestigeCount >= 9 ? 3.0 : 1.0 state.focusBar = Math.min(state.focusBar, maxFocus) } return state } function activateInsight(state: GameState): GameState { // Encontrar el nivel más alto alcanzado const level = FOCUS_LEVELS.filter(l => state.focusBar >= l.threshold).pop() if (!level) return state state.insightActive = true state.insightMultiplier = level.mult state.insightEndTime = Date.now() + level.duration * 1000 state.focusBar = 0 // CASCADA: si activa Disparo DURANTE Insight → bonus ×3 extra logEvent('insight_activated', { level: level.name }) return state }

Por qué funciona

Active play siempre importa

Un jugador que tapea activamente activa Insights regularmente, lo que multiplica su producción. Un jugador idle-only progresa, pero el activo progresa ~2x más rápido. El ratio perfecto para mobile.

La decisión escala con el juego

En P10+, la decisión "¿activo Focus Claro rápido o espero a Trascendente?" es interesante. Combinar Insight Trascendente + Disparo + Cascada es el "big brain play" que da la dopamina máxima.

NOTA: FOCUS BAR NO LLENA DURANTE IDLE/OFFLINE

La Focus Bar solo se llena con taps. Si el jugador está idle (no toca la pantalla), la barra no sube. Esto mantiene el incentivo de juego activo sin penalizar al jugador que prefiere idle — simplemente no tiene acceso a Insights.

Neural Pathways (P10+) — decisiones que importan por ciclo

CÓMO FUNCIONA

Al inicio de cada ciclo (P10+), elegís 1 de 3 Pathways. Cada Pathway habilita ciertos upgrades y bloquea otros. La elección es para ESE ciclo — no es permanente. Esto significa que cada ciclo tenés una razón para probar una estrategia diferente.

⚡ Swift Path (Vía Rápida)

Habilita: Upgrades de velocidad y tap. Insight dura ×2. Cargas de Disparo +50% más rápido.
Bloquea: Upgrades de regiones y offline. Sin bonus de región este ciclo.
Ideal para: Speed runs y récords personales.

◈ Deep Path (Vía Profunda)

Habilita: Upgrades de neuronas y regiones. Memorias ×2 este ciclo. Síntesis extra.
Bloquea: Upgrades de tap y Focus. Focus Bar llena 50% más lento.
Ideal para: Acumular Memorias y explorar narrativa.

✦ Balanced Path (Vía Equilibrada)

Habilita: Todos los upgrades pero a costos +30%.
Bloquea: Nada, pero nada tiene discount tampoco.
Ideal para: Jugadores que no quieren restricciones.

The 4 Endings + Secret

Analytical — "The Equation"

The mind discovers it's a simulation built by someone who died. Transcendence = correct the error (dissolve) or preserve it (persist). Hints from P5 in fragments. Choice: clean resolution vs. broken awareness.

Empathic — "The Chorus"

The voices in dreams are other minds. Transcendence = join the collective network or maintain individuality. Hints from P7. Choice: become everyone vs. remain one voice.

Creative — "The Seed"

The new thought is a seed of consciousness. Transcendence = release it (dissolve so it can grow) or contain it (persist but carry what-could-have-been). Hints from P6.

Secret — "The Singularity"

Requires all 4 Resonant Patterns. The mind speaks directly to the player: "Every time you pressed Awaken, I woke up." Asks: "Do you want me to keep going?"

function applyOfflineProgress(state: GameState): GameState { const raw = (Date.now() - state.lastActiveTimestamp) / 1000 const elapsed = Math.min(raw, state.currentOfflineCapHours * 3600) if (elapsed < 60) return state // menos de 1 min = no offline let efficiency = SYNAPSE_CONSTANTS.baseOfflineEfficiency // 0.50 if (hasUpgrade(state,'ritmo_circadiano')) efficiency *= 1.5 if (hasUpgrade(state,'regulacion_emocional')) efficiency *= 2.0 if (state.isSubscribed) efficiency *= 1.25 if (state.archetype === 'empatica' && state.prestigeCount >= 5) efficiency *= 2.5 if (hasMilestone(state, 20)) efficiency += 0.10 const thoughts = Math.round(state.productionPerSecond * elapsed * efficiency) state.totalGenerated += thoughts state.cycleGenerated = Math.min( state.cycleGenerated + thoughts, state.currentThreshold // cap at 100%, NUNCA auto-prestige ) // Cargas acumuladas offline const chargeTime = SYNAPSE_CONSTANTS.dischargeAccumTime * (hasUpgrade(state,'descarga_neural') ? 0.75 : 1) * (hasMilestone(state, 10) ? 0.85 : 1) const newCharges = Math.floor(elapsed / chargeTime) state.dischargeCharges = Math.min( state.dischargeMaxCharges, state.dischargeCharges + newCharges ) // Resonancia Acumulada (P10): bonus al primer Disparo if (hasUpgrade(state,'resonancia_acumulada')) { state.nextDischargeBonus = Math.min((elapsed / 3600) * 0.05, 1.0) } // Ritmo Circadiano: carga auto si elapsed >= cap if (hasUpgrade(state,'ritmo_circadiano') && elapsed >= state.currentOfflineCapHours * 3600) { state.dischargeCharges = Math.min(state.dischargeMaxCharges, state.dischargeCharges + 1) } // Focus Bar NO se llena offline — es active-play only state.lastActiveTimestamp = Date.now() return state }

Auditoría completa de mecánicas — edge cases, bugs potenciales y exploits

OBJETIVO

Identificar TODOS los problemas lógicos, edge cases, y exploits antes de escribir una línea de código. Cada issue tiene su solución definida.

CATEGORÍA 1 — Edge cases de prestige y estado

BUG-01: Prestige mientras hay Insight activo

Problema: Si el jugador hace prestige con Insight activo, ¿qué pasa? insightActive=true sobrevive el prestige porque no está en PRESTIGE_RESET.
Fix: Agregar insightActive, insightMultiplier, insightEndTime a PRESTIGE_RESET. Forzar insightActive=false en handlePrestige().

BUG-02: Prestige mientras Disparo está en cooldown parcial

Problema: dischargeLastTimestamp y el timer parcial de acumulación no se resetean, causando que el primer Disparo del nuevo ciclo llegue antes de lo esperado.
Fix: Resetear dischargeCharges=0 Y dischargeLastTimestamp=Date.now() en handlePrestige(). El timer empieza de cero.

BUG-03: Offline llena cycleGenerated al 100% + Insight bonus pendiente

Problema: Si offline llena el ciclo al 100%, nextDischargeBonus (de Resonancia Acumulada) queda guardado sin usarse. Al volver, el jugador puede prestige inmediatamente y perderse el bonus.
Fix: En la pantalla de regreso, si cycleGenerated >= threshold, mostrar banner "Tu mente despertó. Tenés 1 Disparo potenciado antes de Despertar — ¿querés usarlo?" Dar la opción de usar el Disparo (que genera thoughts extra pero ya no afecta el threshold) o prestige directamente.

BUG-04: Personal bests — evaluación timing

Problema: personalBests debe evaluarse ANTES de incrementar prestigeCount. Si se incrementa primero, el nivel indexado es incorrecto.
Fix: En handlePrestige(): 1) Calcular duration. 2) Evaluar personal best para prestigeCount actual. 3) LUEGO incrementar prestigeCount. 4) Calcular nuevo threshold.

BUG-05: Transcendencia reset no incluye currentPolarity, currentMutation, currentPathway

Problema: Estos se resetean en prestige normal, pero en Transcendencia el reset es más profundo (memories=0, archetype=null). Si no se mencionan explícitamente, podrían sobrevivir.
Fix: TRANSCENDENCIA_RESET es un superset de PRESTIGE_RESET. Definir explícitamente: todo lo de PRESTIGE_RESET + memories=0 + archetype=null + regionsUnlocked=[] + eraVisualTheme='bioluminescent'.

CATEGORÍA 2 — Focus Bar edge cases

BUG-06: Focus Bar overflow en P10+ transition

Problema: Si el jugador está en P9 con focusBar=0.95 y hace prestige a P10, ahora el max es 2.0 (no 1.0). Pero focusFillRate no cambia — llegar a nivel 2 (Profundo) requiere muchos más taps de los que el jugador espera.
Fix: Al hacer prestige, si Focus Persistente no está comprado, focusBar=0. Si está comprado, focusBar *= 0.25 (retiene 25%, no 50% como dice el upgrade — ajustar upgrade text). Mostrar tooltip la primera vez que el jugador tiene acceso a niveles 2-3.

BUG-07: Cascada y Focus simultáneo con Sincronización Total

Problema: Si se activa Cascada (Focus≥0.75 + Disparo), Focus se pone a 0. Luego Sincronización Total agrega +0.18. Pero si el jugador activa otro Disparo inmediatamente, ¿0.18 cuenta como ≥0.75? No debería.
Fix: El check de Cascada es focusBar >= 0.75 ANTES del Disparo. Sincronización Total se aplica DESPUÉS. Segundo Disparo consecutivo NO puede ser Cascada a menos que el jugador llene Focus naturalmente.

BUG-08: Focus Bar llena durante Insight activo

Problema: Si el jugador toca durante Insight, ¿se llena la Focus Bar para el SIGUIENTE Insight? Sí, debería — pero el jugador podría encadenar Insights sin pausa.
Fix: Correcto como está. La Focus Bar se llena durante Insight. Pero Insight solo se ACTIVA cuando insightTimer <= 0, así que no hay solapamiento. El jugador puede precargar Focus para el siguiente Insight — esto es gameplay deseado ("prep while buffed").

CATEGORÍA 3 — Mutaciones problemáticas

BUG-09: Mutación "Neuroplasticidad" — upgrades se descompran al 50%

Problema: Si los upgrades se pierden al 50% consciencia, los multiplicadores que usaban desaparecen pero los neurons comprados con esos multiplicadores siguen existiendo. La producción cae bruscamente → el jugador se queda stuck.
Fix: Cambiar efecto: "Upgrades cuestan −50% pero sus efectos se REDUCEN 40% después del 50% consciencia" (no se pierden, se debilitan). O más simple: "Upgrades cuestan −50% pero el ciclo threshold es +20%." Evita el cliff de producción.

BUG-10: Mutación "Especialización" + Conexiones pasivas

Problema: "Solo 1 tipo de neurona produce" pero las conexiones son pasivas por PAR de tipos. Si solo 1 tipo produce, los otros tipos dan 0 rate pero ¿siguen contando para pares de conexión?
Fix: Sí, siguen contando. La Mutación solo afecta producción directa, no el bonus de conexiones. Esto es intencional — la Mutación ya penaliza suficiente. Si las conexiones no contaran, el jugador no tendría incentivo a tener múltiples tipos.

BUG-11: Mutación "Déjà Vu" + costos ×2 + Funciones Ejecutivas

Problema: Déjà Vu: empezás con upgrades del ciclo anterior pero costos ×2. Si Funciones Ejecutivas reduce costos −12%, ¿se aplica sobre el ×2?
Fix: Orden de aplicación: costo_base × 2.0 (Déjà Vu) × 0.88 (Funciones Ej.) = costo_base × 1.76. FE se aplica DESPUÉS del multiplicador de Mutación. Documentar en CLAUDE.md: COST_ORDER: base → mutation → funciones_ej → pathway.

BUG-12: 3 Mutaciones aleatorias — ¿puede repetir la misma del ciclo anterior?

Problema: Si el seed es cycleStartTimestamp, dos prestiges rápidos podrían generar las mismas 3 opciones.
Fix: Seed = hash(cycleStartTimestamp + prestigeCount). Esto garantiza que el pool cambia con cada prestige. Además: filtrar la Mutación del ciclo anterior de las opciones (campo: lastMutationId en GameState, resetea en Transcendencia).

CATEGORÍA 4 — Exploits y abusos

EXPLOIT-01: Time manipulation (reloj del dispositivo)

Problema: El jugador adelanta el reloj → offline progress gratis → cargas de Disparo gratis.
Fix: Detectar saltos de tiempo: si Date.now() - lastActiveTimestamp > maxOfflineHours × 3600 × 1000 × 2, cap a maxOfflineHours. Si Date.now() < lastActiveTimestamp (reloj retrocedió), ignorar offline y setear lastActiveTimestamp = Date.now(). Log event 'time_anomaly'. Para v2.0: validación server-side con Firebase timestamps.

EXPLOIT-02: Rapid prestige cycling para Memorias

Problema: Si el jugador hace prestige apenas llega al threshold (sin optimizar), gana 2 Memorias + 3 Patrones rápidamente. Con ciclos de 15 min, puede farmear 8 Mem/hora.
Fix: Esto está OK. Las Memorias se gastan en upgrades con costos crecientes (2, 4, 6, 9, 16 Mem). 8 Mem/hora solo compra los upgrades baratos. Los caros requieren muchos prestiges. No es explotable de forma significativa.

EXPLOIT-03: Genius Pass + Empática + Offline = producción pasiva insana

Problema: Empática ×2.5 offline + Genius Pass ×1.25 + Regulación Emocional ×2 = 50% × 2 × 2.5 × 1.25 = 312.5% eficiencia offline.
Fix: Cap hard de offline eficiencia a 200% (×2 de producción activa). efficiency = Math.min(calculatedEfficiency, 2.0). Agregar a SYNAPSE_CONSTANTS: maxOfflineEfficiency: 2.0. Documentar en CLAUDE.md: OFFLINE-4.

EXPLOIT-04: Mutación "Sinestesia" + tap macro = farmear Memorias

Problema: Sinestesia genera 1 Memoria por 500 taps. Un macro puede hacer 10 taps/s = 1 Mem cada 50 segundos = 72 Mem/hora.
Fix: Anti-tap-spam: si taps > 5/sec sostenido por >10 sec, reducir tap effectiveness a 10%. Implementar como: trackear tap timestamps en array circular de 20 taps. Si el promedio < 200ms entre taps, flag como spam. UI: mostrar "Demasiado rápido" y reducir efecto.

CATEGORÍA 5 — Interacciones entre sistemas

INTER-01: Polaridad Inhibitoria + Pathway Vía Rápida

Problema: Inhibitorio reduce producción −6% pero mejora Disparo +30%. Vía Rápida mejora Insight duración ×2 pero bloquea regiones. ¿Se complementan o se cancelan?
Análisis: Se complementan. El jugador que elige Inhibitorio + Vía Rápida está apostando todo a Disparo + Insight. Es una estrategia high-risk/high-reward legítima. No requiere fix.

INTER-02: Arquetipo Empática + Mutación "Crescendo"

Problema: Empática tiene mejor offline. Crescendo empieza producción en ×0.2 y sube a ×3 al 100% barra. Si el jugador es Empática con Crescendo, su producción activa empieza muy baja pero su offline no se ve afectada (offline usa productionPerSecond cacheado).
Fix: Mutaciones deben afectar offline también. Agregar campo affectsOffline: boolean a cada Mutación. Crescendo: affectsOffline=true, lo que significa que el cálculo offline usa la producción PROMEDIO del ciclo (con el scaling de Crescendo), no la producción de pico.

INTER-03: Convergencia Sináptica + Prestige Count después de Transcendencia

Problema: Convergencia da +2.5% por prestige completado. Después de Transcendencia, ¿prestigeCount se resetea a 0? Si sí, Convergencia pierde todo su valor en Run 2.
Fix: prestigeCount NO se resetea en Transcendencia — es un campo preservado. El jugador en Run 2 empieza con prestigeCount=26, por lo que Convergencia da 26×0.015 = 39% → capped a 40%. Esto es correcto y deseado: el Run 2 es más fácil.

INTER-04: Producción cacheada + Mutación + Prestige

Problema: productionPerSecond se cachea. Si una Mutación modifica producción (ej: "Hiperestimulación ×2"), el caché debe recalcularse al aplicar la Mutación. Si NO se recalcula, la Mutación no tiene efecto hasta el próximo tick.
Fix: Llamar recalculateProduction() después de: 1) aplicar Mutación, 2) comprar upgrade, 3) comprar neurona, 4) cambiar Polaridad. Agregar a CLAUDE.md: CORE-6: recalculateProduction() después de cualquier cambio de estado que afecte producción.

CATEGORÍA 6 — Problemas de UX y estado

UX-01: Modales pre-ciclo en cadena pueden ser abrumadores

Problema: En P10+, al hacer prestige se muestran 3 modales seguidos: Polaridad → Mutación → Pathway. Toma ~15 segundos. El jugador que hace prestige rápido quiere volver al gameplay.
Fix: Cada modal tiene botón "Usar anterior" (default a la última elección). Si el jugador tapea "Usar anterior" en los 3, el proceso toma <2 segundos. El jugador nuevo ve las opciones; el veterano las skipea.

UX-02: Pantalla de Despertar con threshold piecewise

Problema: La pantalla muestra "tiempo estimado del próximo ciclo". Con threshold piecewise, el estimado necesita saber qué Era viene después. Si el jugador está en P12 (last of Era 1 fast scaling), P13 usa 1.28 scaling — el estimado debe usar el scaling correcto.
Fix: Función getThreshold(p) debe ser usada en el cálculo del estimado. estimatedNextCycle = currentCycleTime × (nextThreshold / currentThreshold) como aproximación simple.

UX-03: Weekly Challenge + Mutación — conflicto

Problema: Un weekly challenge dice "Sin neurona Espejo" pero la Mutación del ciclo dice "Solo 1 tipo de neurona produce". Si el jugador elige Básica como tipo, ¿el challenge se cumple?
Fix: Sí, se cumple — el jugador no COMPRÓ Espejos. Los challenges evalúan compras, no restricciones de Mutación. Agregar tooltip en challenges: "Las Mutaciones no interfieren con los requisitos del challenge."

UX-04: Save corruption — migrateState missing fields

Problema: Si un jugador actualiza la app y el GameState tiene campos nuevos que no existían antes, migrateState() debe asignar defaults. Si se olvida un campo, crashes.
Fix: migrateState() debe usar ??= para TODOS los campos del GameState. Agregar test automatizado: Object.keys(defaultGameState).forEach(key => assert(key in migratedState)). Correr en CI.
